## 개념 정리

## React Core (리액트 기본 엔진)

#### React

- 상태(state)에 따라 화면을 자동 갱신하는 UI 라이브러리

#### JSX

- JS 안에서 HTML처럼 작성하는 문법

#### Component

- 재사용 가능한 화면 블록

#### Rendering

- state/props 변경 시 화면 다시 그림

#### Virtual DOM

- 실제 DOM 전에 비교해서 변경 부분만 반영

## Data Flow (데이터 전달 구조)

#### State

- 컴포넌트 내부에서 관리하는 화면 데이터

#### Props

- 부모 → 자식으로 전달되는 값

#### Lifting State Up

- 여러 컴포넌트가 쓰는 state를 부모로 올림

## React Hooks (전부 Hook)

#### useState

- 상태 생성 및 변경

#### useEffect

- 컴포넌트 생성/업데이트 시 실행 (API 호출)

#### useReducer

- 복잡한 상태를 규칙 기반으로 관리

#### useRef

- 렌더링 없이 값 저장

#### useContext

- 전역 데이터 공유

#### Custom Hook

- 반복 로직을 하나의 Hook으로 묶음

## Rendering Control (화면 출력 방식)

#### Conditional Rendering

- 조건에 따라 컴포넌트 표시

#### List Rendering

- 배열 map으로 화면 출력

#### key

- 리스트 요소 구분자

#### Fragment

- 불필요한 div 제거

## Performance Optimization (성능)

#### useMemo

- 계산 결과 저장

#### useCallback

- 함수 재생성 방지

#### React.memo

- props 안 변하면 렌더링 차단

#### Memoization

- 이전 계산 결과 재사용

## Server Communication (통신 – React 외부)

#### axios

- API 통신 라이브러리
- [링크](https://axios-http.com/kr/docs/intro)

#### interceptor

- 요청/응답 중간 처리 (토큰, 에러)

## Application Structure (앱 구조)

#### SPA

- SPA = Single Page Application
- 새로고침 없이 컴포넌트만 교체하는 웹 구조

#### Routing

- URL로 컴포넌트 변경

#### Controlled Component

- input 값을 state로 관리

#### Uncontrolled Component

- DOM이 직접 값 관리

## Error / Loading Handling

#### Error Boundary

- 에러 발생 시 앱 보호

#### Lazy

- 컴포넌트 지연 로딩

#### Suspense

- 로딩 UI 제어

## Global State

#### Context API

- 전역 상태 관리

## Framework Layer

#### Next.js

- React 기반 풀스택 프레임워크

#### Server Components

- 서버에서 렌더링

#### Hydration

- 서버 HTML과 클라이언트 연결
