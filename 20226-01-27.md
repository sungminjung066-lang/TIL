## 개념 정리

### React

- UI를 컴포넌트 단위로 만들고 상태 변화에 따라 화면을 자동 업데이트하는 라이브러리

### Component

- 화면을 구성하는 재사용 가능한 블록

### JSX

- JS 안에서 HTML처럼 작성하는 문법

### State

- 화면 변경의 기준이 되는 데이터

### Props

- 부모 컴포넌트가 자식에게 전달하는 값

### Virtual DOM

- 실제 DOM 전에 가짜 DOM으로 비교해서 바뀐 부분만 렌더링

### Rendering

- state/props 변경 시 화면을 다시 그리는 과정

## Hook

### useState

- 컴포넌트 안에서 상태(state)를 만들고 변경하는 훅

### useEffect

- 컴포넌트가 뜨거나 업데이트될 때 실행되는 함수

### useRef

- 값은 유지하지만 화면 렌더링에는 영향 안 주는 저장소

### useContext

- props 없이 전역 데이터 공유

### useMemo

- 계산 결과를 기억해서 불필요한 재계산 방지

### useCallback

- 함수 재생성을 막아 성능 최적화

## 기타

### SPA

- 페이지 새로고침 없이 화면만 바뀌는 웹

### Routing

- URL에 따라 컴포넌트를 바꾸는 것

### Controlled Component

- input 값을 state로 관리하는 방식

### Uncontrolled Component

- DOM이 직접 값 관리하는 방식

### Lifting State Up

- 여러 컴포넌트가 쓰는 state를 부모로 올리는 것

### axios

- 서버랑 통신(API 요청)하기 쉽게 만든 라이브러리

### interceptor

- API 요청/응답을 중간에서 가로채 공통 작업 처리
