## 1. 이벤트

- JS를 통해서 HTML 엘리먼트를 가져오는 법

- 가져온 엘리먼트(JS 객체)에 이벤트 부여

## 2. 동기 / 비동기

#### 동기(적으로 행동한다?)

쿠팡에서 사과를 주문했을 경우,
사과가 도착할 때까지 문앞에서 기다리고 있음

엄마가 지나가다가 뭐하니?
"사과 기다리는 중인데요?"

엄마: 하아....

사과가 도착할 때까지 다른 작업을 수행하지 않음

나?라는 사람을 프로그래밍 코드로 가져오면
JS 프로그램 실행의 제어권을 가지고 있는 '스레드(Thread)'이 차단된(Blocking) 상태

- 나? - 하나의 스레드
  스레드가 코드를 1번 라인부터 ~ 20번 라인까지 실행하다가...
  사과를 주문하고 기다리는 함수()를 만났더니 거기서 실행을 멈춤
  따라서 21번 라인부터 실행을 못하고 있는 것

- 스레드(Thread) - 프로그램 실행의 제어권을 가지고, 실제 실행을 담당하는 역할, 객체
  -> 스레드는 프로세스 내에서 실제 코드의 실행을 담당하는 역할
  프로세스 내에 스레드가 1개인 프로그램(싱글 스레드)
  프로세스 내에 스레드가 2개 이상인 프로그램(멀티 스레드)

- 프로세스(Process) - 그 프로그램을 메모리 상에 올려서 실행시키는 단위(프로세스)
- 프로그램(Program) - 디스크 상 어딘가에 위치한 소스코드 묶음
  ex. 제가 현재 실행 중인 Chrome 브라우저

PC를 구매했음
Chrome도 안깔려있음

Chrome이라는 프로그램을 C드라이브같은 디스크에 설치함
(Chrome이라는 브라우저는 소프트웨어이고, 소프트웨어는 소스코드로 작성되어 있음)

이제 사용자가 Chrome을 더블클릭하여 실행을 했음
-> Windows기준으로, OS가 디스크에 있는 Chrome 프로그램을 찾아서 프로세스의 형태로 실행시킴
프로세스는 메모리에 할당되어 동작함

그러다가 Chrome을 종료하면, 프로세스가 종료됨

#### 비동기

쿠팡에서 사과를 주문했을 경우,
사과가 도착할 때까지 딴짓하고 있으면 됨
(유튜브를 봐도 되고, 넷플릭스도 되고 등등)

쿠팡맨께서 똑똑하면 (어? 사과가 도착햇네?? 사과 먹으면됨)

빨간불 신호가 걸리면 휴대폰을 열고 카톡을 함
뒤 차가 빵빵하면 앞으로 감

나? 라는 사람(뇌를 스레드)

JS는 기본적으로 스레드가 하나로 동작함
동기적으로 동작할 수 밖에 없는 한계를 가지고 있다.

JS 런타임 자체는 싱글 스레드
JS는 어디서 동작한다??? 브라우저에서 동작한다

브라우저는 하나의 소프트웨어
브라우저가 동작할 때는 브라우저만의 별도의 백그라운드 스레드들이 별도로 있음

JS 런타임 자체는 싱글스레드이지만, 비동기적으로 동작할 수 있는 이유는 브라우저가 가진 별도의 스레드(Web API)들도 같이 동작하기 때문에 비동기적으로 처리할 수 있음
